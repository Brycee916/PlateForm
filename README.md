# Plateform Builder MVP

This repo now includes a working full-stack MVP for a restaurant website builder:

- Owner authentication (register/login/logout)
- Restaurant onboarding flow
- Content editor (brand, hours, address, hero text)
- Menu CMS (categories + dishes)
- Theme editor (colors + gallery URLs)
- One-click publish action
- Deployment history
- Generated static restaurant site output in `published/<slug>/index.html`

## Stack

- Frontend: React + Vite + Tailwind
- Backend: Node HTTP server (`server/index.js`) with JSON persistence
- Persistence: `data/db.json` (auto-created)
- Publish output: static HTML generated by `server/siteRenderer.js`

## Run locally

1. Install dependencies:

```bash
npm install
```

2. Start backend (Terminal 1):

```bash
npm run dev:server
```

3. Start frontend (Terminal 2):

```bash
npm run dev
```

4. Open the frontend URL shown by Vite (usually `http://localhost:5173`).

## Environment variables (optional)

- `PORT` for backend port (default `8787`)
- `CORS_ORIGIN` for frontend origin (default `http://localhost:5173`)
- `VITE_API_URL` for frontend API base (default `http://localhost:8787`)

## API overview

- `POST /api/auth/register`
- `POST /api/auth/login`
- `POST /api/auth/logout`
- `GET /api/me`
- `GET /api/restaurants`
- `POST /api/restaurants`
- `PUT /api/restaurants/:id`
- `POST /api/restaurants/:id/publish`
- `GET /api/restaurants/:id/deployments`
- `GET /api/public/:slug`
- `GET /sites/:slug` (serves published HTML)

## Project structure

- `src/App.jsx`: dashboard UI + onboarding + editor
- `server/index.js`: backend API + auth + publish orchestration
- `server/siteRenderer.js`: static site HTML renderer
- `data/db.json`: persisted app data (auto-generated)
- `published/`: generated deployed sites

## Notes

This is a strong MVP foundation. For production readiness, the next upgrades are:

- Replace JSON DB with Postgres (Supabase/Neon/etc.)
- Replace in-file sessions with JWT + refresh token strategy
- Real deployment target (Vercel/Cloudflare Pages/S3)
- Custom domain automation via DNS provider API
- Stripe billing and plan limits
- Role-based team access and audit logs
- Automated tests + observability
